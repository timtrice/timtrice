Delete Duplicate Emails

Dec 11, 2017 3 min read SQL, LeetCode

Write a SQL query to delete all duplicate email entries in a table named Person, keeping only unique emails based on its smallest Id.

+----+------------------+
| Id | Email            |
+----+------------------+
| 1  | john@example.com |
| 2  | bob@example.com  |
| 3  | john@example.com |
+----+------------------+

Id is the primary key column for this table.

For example, after running your query, the above Person table should have the following rows:

+----+------------------+
| Id | Email            |
+----+------------------+
| 1  | john@example.com |
| 2  | bob@example.com  |
+----+------------------+
CREATE DATABASE IF NOT EXISTS `LeetCode`;
USE `LeetCode`;
CREATE TABLE IF NOT EXISTS `Person` (
  `Id` INT, 
  `Email` VARCHAR(255)
);
TRUNCATE TABLE `Person`;
INSERT INTO `Person` 
  (`Id`, `Email`) 
VALUES 
  (1, "john@example.com"), 
  (2, "bob@example.com"), 
  (3, "john@example.com");
SELECT * 
FROM `Person`;
Table 1: 3 records
Id Email
1 john@example.com
2 bob@example.com
3 john@example.com

Solution

This query is a bit tricky. There are several ways we can find duplicates; my first reaction is always to do a COUNT. This does not tell us the Id of duplicate emails, though. We have to GROUP Email and since Id is different for each Email, MySQL will generate an error like the following:

Error in query (1055): Expression #1 of SELECT list is not in GROUP BY clause and contains nonaggregated column ‘LeetCode.Person.Id’ which is not functionally dependent on columns in GROUP BY clause; this is incompatible with sql_mode=only_full_group_by

This is the single-value rule implementation in the more recent versions of MySQL (error message may vary depending on your SQL server).

A key is given in the description:

keeping only unique emails based on its smallest Id.

Smallest = MIN.

We can select MIN(Id) for each Email group.

This is where I got tripped up a little bit. If I run a query to find the minimum Id of each group, I get results I expect:

SELECT
  MIN(`Id`) AS `min_id`
FROM `Person`
GROUP BY `Email`;
Table 2: 2 records
min_id
2
1

From here, I felt I could just run this query:

DELETE
FROM `Person`
WHERE `Id` NOT IN (
  SELECT
    MIN(`Id`) AS `min_id`
  FROM `Person`
  GROUP BY `Email`
);

This, however, will not work:

Error in query (1093): You can’t specify target table ‘Person’ for update in FROM clause

For my version of MySQL, this is documented in 13.2.11 UPDATE Syntax; in short,

You cannot update a table and select from the same table in a subquery.

I did find two good examples of overcoming this obstacle:

  • MySQL Error 1093 - Can’t specify target table for update in FROM clause

  • How to resolve MySQL error “You can’t specify target table X for update in FROM clause”? [duplicate]

In effect, I have to bury my query for min_id within another SELECT query; with that query I can then run my DELETE statement to remove any Id that is not in min_id.

DELETE 
FROM `Person`
WHERE `Person`.`Id` NOT IN (
  SELECT * 
  FROM (
    SELECT
      MIN(`Id`) AS `min_id`
    FROM `Person`
    GROUP BY `Email`) AS `t1`);

Being around 79% isn’t bad except that it tells me there’s quite a bit of room to improve. For the time being, I’ve added this challenge to my list to improve at a later time.
