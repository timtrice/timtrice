Rising Temperature

Dec 18, 2017 3 min read SQL, LeetCode

Given a Weather table, write a SQL query to find all dates’ Ids with higher temperature compared to its previous (yesterday’s) dates.

+---------+------------+------------------+
| Id(INT) | Date(DATE) | Temperature(INT) |
+---------+------------+------------------+
|       1 | 2015-01-01 |               10 |
|       2 | 2015-01-02 |               25 |
|       3 | 2015-01-03 |               20 |
|       4 | 2015-01-04 |               30 |
+---------+------------+------------------+

For example, return the following Ids for the above Weather table:

+----+
| Id |
+----+
|  2 |
|  4 |
+----+

Solution

SET @prevtemp := NULL;
SET @prevdate := NULL;

SELECT `Id`
FROM (
  SELECT
    CASE
      WHEN `Temperature` > @prevtemp
        AND DATE_SUB(`Date`, INTERVAL 1 DAY) = @prevdate
      THEN `Id`
      ELSE NULL
    END AS `Id`,
    @prevtemp := `Temperature`,
    @prevdate := `Date`
  FROM `Weather`
  ORDER BY `Date` ASC) AS `t1`
WHERE `Id` IS NOT NULL;

Explanation

MySQL 8+ implements new functions LAG() and LEAD() which would make this query much easier to write. I’m using version 5.7.20, though, so these new functions are not an option for me.

The alternative is using variables @prevtemp and @prevdate.

The solution uses a subquery to return Id only if Temperature is greater than @prevtemp and Date is less than @prevdate; otherwise, it returns NULL.

Once this condition is first tested, the variables @prevtemp and @prevdate (originally assigned NULL) are assigned the current Temperature and Date values, respectively.

My original query took a similar approach:

SET @prevtemp = NULL;

SELECT `Id`
FROM (
  SELECT 
    `Id`, 
    `Date`, 
    `Temperature`, 
    @prevtemp AS `prevtemp`, 
    @prevtemp := `Temperature` AS `currtemp`
  FROM `Weather`
  ORDER BY `Date`) AS `t1`
WHERE `prevtemp` < `currtemp`;

This worked for the original test case but failed on gaps between dates.

Yet another approach was doing an INNER JOIN on the table.

SELECT `t1`.`Id`
FROM `Weather` AS `t1`
INNER JOIN `Weather` AS `t2`
WHERE `t1`.`Temperature` > `t2`.`Temperature`
AND `t1`.`Date` = DATE_ADD(`t2`.`Date`, INTERVAL 1 DAY);

This query passed all tests but was horribly slow (it only beat 3% of submissions - ugly). Of the 14 test cases, it took 2083ms to run where the solution posted above only took 1069ms, a 51.3% reduction.

Cleanup

DROP TABLE `Weather`;
