Combine Two Tables

Using LEFT JOIN to combine tables in MySQL, MariaDB

Join two tables in a SQL database to create one table.

USE `employees`;

Table A (employees) has a PRIMARY KEY emp_no.

Table B (titles) also has a PRIMARY KEY emp_no.

Having similar-named primary keys is not required for JOIN operations.

A LEFT JOIN statement will return all rows from Table A along with matching rows from Table B. The ON statement identifies how the tables are joined. In this example, this is the emp_no field.

Return employees and titles joined as one table, limited to the first ten values.

SELECT * 
FROM `employees`
LEFT JOIN `titles`
ON `employees`.`emp_no` = `titles`.`emp_no` 
LIMIT 0, 10;

emp_no 10004 contains two values:

SELECT `employees`.`emp_no`, 
  `employees`.`first_name`, 
  `employees`.`last_name`, 
  `titles`.`title`
FROM `employees`
LEFT JOIN `titles`
ON `employees`.`emp_no` = `titles`.`emp_no` 
WHERE `employees`.`emp_no` = 10004;
Table 5: 2 records
emp_no first_name last_name title

This employee has two entires in titles, therefore exists in our joined table twice. If employees.emp_no does not exist in titles.emp_no, then our joined table would return the value of employees.emp_no, but titles.title would be NA.

To find these values we must create a subquery (our original query). We cannot simply do a search where titles.title is NULL; such values would not be related to our query.

By building a SELECT statement based on the subquery, where titles would return NA when no matches are found, we can find these mismatches.

SELECT CASE WHEN COUNT(*) > 0 THEN COUNT(*) ELSE "NULL" END AS `Mismatches`  
FROM (SELECT `employees`.`emp_no`, 
    `employees`.`first_name`, 
    `employees`.`last_name`, 
    `titles`.`title`
  FROM `employees`
  LEFT JOIN `titles`
  ON `employees`.`emp_no` = `titles`.`emp_no`) `t`
WHERE `title` IS NULL;
