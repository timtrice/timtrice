Finding nth salaries

Find ranked salaries in employees database

We are asked to find specific salary.salaries values within our employees database.

USE `employees`;

Let’s find who has the highest salaries in our database.

SELECT * 
FROM `salaries` 
ORDER BY `salaries`.`salary` DESC 
LIMIT 0, 5;
Table 1: 5 records
emp_no salary from_date to_date
43624 158220 2002-03-22 9999-01-01
43624 157821 2001-03-22 2002-03-22
254466 156286 2001-08-04 9999-01-01
47978 155709 2002-07-14 9999-01-01
253939 155513 2002-04-11 9999-01-01

We notice in our query above we have multiple salaries per employee (see emp_no 43624). This is because salaries is a historical table; it keeps track of each employees salary over time. So we must find the latest salary per employee.

SELECT *, 
  MAX(`to_date`) AS `max_date`
FROM `salaries`
GROUP BY `emp_no`
LIMIT 0, 5;

This generates an error:

ERROR 1055 (42000): Expression #2 of SELECT list is not in GROUP BY clause and contains nonaggregated column ‘employees.salaries.salary’ which is not functionally dependent on columns in GROUP BY clause; this is incompatible with sql_mode=only_full_group_by

This error is effectively telling us that our query violated the “Single-Value Rule”. Every column we are selecting must have the same value per row group. In our case - having distinct values, i.e., salary, from_date, to_date - requires the database server to make assumptions on which of the other column values we wish to return.

Isolate on the first three rows of emp_no 10001:

SELECT * 
FROM `salaries` 
WHERE `emp_no` = 10001
LIMIT 0, 3;
Table 2: 3 records
emp_no salary from_date to_date
10001 60117 1986-06-26 1987-06-26
10001 62102 1987-06-26 1988-06-25
10001 66074 1988-06-25 1989-06-25

I GROUP BY emp_no then find MAX(to_date). But, what salary value will the databse return? What from_date value will the server return? It is ambiguous and violates SQL standards.

If I specify my query to return emp_no and MAX(to_date) then my query will execute correctly.

SELECT `emp_no`, 
  MAX(`to_date`) 
FROM `salaries` 
GROUP BY `emp_no` 
LIMIT 0, 5;
Table 3: 5 records
emp_no MAX(to_date)
10001 9999-01-01
10002 9999-01-01
10003 9999-01-01
10004 9999-01-01
10005 9999-01-01

This query would solve our antipattern, generally speaking. But, it lacks the data we need; current salary. To resolve this we can use the query in Table 3 as a derived table and build a JOIN from this table.

SELECT `t1`.`emp_no`, 
  `t1`.`salary`, 
  `t1`.`to_date`
FROM `salaries` AS `t1` 
JOIN (
  SELECT `emp_no`, 
    MAX(`to_date`) AS `max_to_date` 
  FROM `salaries` 
  GROUP BY `emp_no`) AS `t2` 
ON `t1`.`emp_no` = `t2`.`emp_no` 
  AND `t1`.`to_date` = `t2`.`max_to_date` 
ORDER BY `t1`.`salary` DESC 
LIMIT 0, 5;

43624 158220 9999-01-01
254466 156286 9999-01-01
47978 155709 9999-01-01
253939 155513 9999-01-01
109334 155190 9999-01-01

This query still doesn’t guarantee we have only one salary per emp_no. If you haven’t validated your data you must assume there could potentially be some issues. Trust, but verify. For the current purposes, we’ll assume our data is accurate.

Currently, we’re returning the top five salaries in our database. However, we have no names. Let’s add them. To do this, we JOIN the employees table ON emp_no.

SELECT `t1`.`emp_no`, 
  `employees`.`emp_no`, 
  `employees`.`first_name`, 
  `employees`.`last_name`, 
  `t1`.`salary`, 
  `t1`.`to_date`
FROM `salaries` AS `t1` 
JOIN (
  SELECT `emp_no`, 
    MAX(`to_date`) AS `max_to_date` 
  FROM `salaries` 
  GROUP BY `emp_no`) AS `t2` 
ON `t1`.`emp_no` = `t2`.`emp_no` 
  AND `t1`.`to_date` = `t2`.`max_to_date` 
LEFT JOIN `employees` 
ON `t1`.`emp_no` = `employees`.`emp_no` 
ORDER BY `t1`.`salary` DESC 
LIMIT 0, 5;

At this point, you can modify the LIMIT call to return all or only specific ranks (keep in mind there are no tie-breakers in this query).

The last result set may look fine from a developer’s perspective. Let’s clean it up though, remembering that our data will become part of a report. We can CONCAT first_name and last_name, FORMAT salary, and drop emp_no and to_date as they are not needed.

SELECT CONCAT(`employees`.`last_name`, ", ", `employees`.`first_name`) AS `Name`, 
  CONCAT("$", FORMAT(`t1`.`salary`, 0)) AS `Salary`
FROM `salaries` AS `t1` 
JOIN (
  SELECT `emp_no`, 
    MAX(`to_date`) AS `max_to_date` 
  FROM `salaries` 
  GROUP BY `emp_no`) AS `t2` 
ON `t1`.`emp_no` = `t2`.`emp_no` 
  AND `t1`.`to_date` = `t2`.`max_to_date` 
LEFT JOIN `employees` 
ON `t1`.`emp_no` = `employees`.`emp_no` 
ORDER BY `t1`.`salary` DESC 
LIMIT 0, 5;
