Create A Database

Creating a database is one of the simplest SQL statements. There are no parameters required other than the name of the database.

CREATE DATABASE `sakila`;

I wrap sakila in backticks only for highlighting. While I would encourage everyone to do so for code legibility, MariaDB and MySQL do not require the use of backticks or single/double quotes.

If you attempt to create a database that already exists then you will receive an error:

MySQL Error 1007 ERROR 1007 (HY000): Can’t create database sakila; database exists

So we can add an IF NOT EXISTS clause to reduce the error to a warning:

CREATE DATABASE IF NOT EXISTS `sakila`;

Show Warnings

The previous SQL statement prevents the ERROR 1007 code but we still get a warning. To review any warnings from a statement:

SHOW WARNINGS;
+-------+------+-------------------------------------------------+
| Level | Code | Message                                         |
+-------+------+-------------------------------------------------+
| Note  | 1007 | Can't create database 'sakila'; database exists |
+-------+------+-------------------------------------------------+

Show Databases

This is how we get the names of all databases in a setup:

SHOW DATABASES;
+--------------------+
| Database           |
+--------------------+
| sakila             |
| mysql              |
| performance_schema |
+--------------------+

Drop A Database

Sometimes you may want to completely remove a database. This also is a very simple statement:

DROP DATABASE `sakila`;

If sakila does not yet exist MySQL will generate an error:

MySQL Error 1008 ERROR 1008 (HY000): Can’t drop database ‘sakila’; database doesn’t exist

Again, we add an IF clause:

DROP DATABASE IF EXISTS `sakila`;

MariaDB and MySQL will first check to see if the database exists before proceeding with the DROP statement.

Dropping databases and tables cannot be undone unless the data files have been backed up elsewhere, even on master/slave setups.

Use Database

Creating a database does not automatically make any proceeding SQL statements use that database. For that we can set the database:

USE `sakila`;

Now any additional statements or queries we execute will use sakila. You do not have to use the USE statement; you can append your database name to your table name in every statement. I personally go with USE and I am not aware of traditional conventions that say otherwise. This makes any statements and queries a bit easier to read as it is less code. But it could potentially become ambiguous and lead to confusion. Whatever project you are working on, just make sure the rules are clearly identified in procedures and that you remain consistent.
